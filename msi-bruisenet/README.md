**MSI-BruiseNet** — A lightweight deep learning framework for semantic segmentation of apple bruises using 9-band snapshot multispectral imaging (MSI).

---

## Abstract

Apple bruise detection is critical for post-harvest quality control. This project implements **MSI-BruiseNet**, a UNet-based semantic segmentation network specifically designed for 9-band near-infrared multispectral images (713–920 nm). The architecture introduces **Local Spectral Anomaly Attention (LSAA)**, a module that exploits local spectral residuals to highlight bruise-indicative anomalies, and **ConvGLU**, a gated fusion mechanism for encoder–decoder skip connections. Built on a MobileNetV2 backbone adapted for 9-channel input, the model is lightweight enough to train and deploy on an edge device with a single RTX 3060 Laptop GPU (6 GB VRAM). The codebase supports 5-fold cross-validation with multiple random seeds, systematic ablation studies (LSAA vs SE vs CBAM vs ECA), area-stratified evaluation, and spectral pre-analysis.

## Architecture

```
Input: (B, 9, H, W)  <-- 9-channel MSI .npy
  |
  +-- MobileNetV2 Encoder (first conv: 9 -> 32 channels, ImageNet pretrained)
  |     Outputs 4-level features: F1(1/2), F2(1/4), F3(1/8), F4(1/16)
  |
  +-- At each skip connection:
  |     F_skip = ConvGLU( LSAA(F_encoder), F_decoder_up )
  |
  |     LSAA (Local Spectral Anomaly Attention):
  |       F_bar  = AvgPool_k(F_e)         -- local background estimate
  |       R_s    = F_e - F_bar             -- spectral residual
  |       W_a    = Sigmoid(BN(Conv3x3(ReLU(BN(Conv1x1(R_s))))))
  |       F_lsaa = F_e * W_a + F_e        -- modulation + bypass
  |
  |     ConvGLU (Gated Fusion):
  |       G     = Sigmoid(Conv3x3(Cat(F_lsaa, F_d)))
  |       F_out = G * F_lsaa + (1 - G) * F_d
  |
  +-- UNet Decoder (bilinear upsample + skip fusion + 2x Conv-BN-ReLU)
  |
  +-- 1x1 Conv -> (B, 2, H, W) logits (Background / Bruise)
```

### Image Format (.npy)

| Property | Value |
|----------|-------|
| Path | `data/images/` |
| Format | NumPy `.npy` |
| Shape | `(H, W, 9)` — height x width x 9 bands |
| dtype | `float32` or `uint16` |
| Bands | 713, 736, 759, 782, 805, 828, 851, 874, 897, 920 nm |
| Naming | `sample_001.npy`, `sample_002.npy`, ... |

### Mask Format (.npy)

| Property | Value |
|----------|-------|
| Path | `data/masks/` |
| Format | NumPy `.npy` |
| Shape | `(H, W)` — same H, W as the corresponding image |
| dtype | `uint8` |
| Values | `0` = background, `1` = bruise |
| Naming | Must match the corresponding image name exactly |

```json
{
  "fold_0": {"train": ["sample_001", "sample_003"], "val": ["sample_002"]},

  ...
}
```

## Configuration

All hyperparameters are managed in `configs/config.yaml`. Key fields users should modify:

| Field | Description | Default |
|-------|-------------|---------|
| `data.image_dir` | Path to .npy images | `data/images/` |
| `data.mask_dir` | Path to .npy masks | `data/masks/` |
| `data.input_size` | Training resize target | `512` |
| `train.batch_size` | Batch size (lower if OOM) | `4` |
| `train.epochs` | Total training epochs | `200` |
| `model.attention` | Attention module | `lsaa` |
| `evaluation.pixel_per_cm` | Pixel-to-cm calibration | `50.0` |

## Quick Start

```bash
# Step 0 (optional): Generate dummy data for smoke testing
python scripts/generate_dummy_data.py --config configs/config.yaml

# Step 1: Compute per-channel normalisation statistics
python scripts/compute_norm_stats.py --config configs/config.yaml

# Step 2: Generate 5-fold cross-validation splits
python scripts/prepare_splits.py --config configs/config.yaml

# Step 3 (optional): Spectral pre-analysis
python utils/spectral_analysis.py --config configs/config.yaml
# Or use the Jupyter notebook: notebooks/spectral_preanalysis.ipynb

# Step 4: Train (5 folds x 3 seeds = 15 runs)
python scripts/train.py --config configs/config.yaml

# Step 5: Evaluate a specific checkpoint
python scripts/evaluate.py --config configs/config.yaml \
    --checkpoint outputs/checkpoints/fold0_seed42_best.pth --fold 0
```

### Resume Training

```bash
python scripts/train.py --config configs/config.yaml \
    --resume outputs/checkpoints/fold0_seed42_best.pth
```

### Override Config from CLI

```bash
python scripts/train.py --config configs/config.yaml \
    --override model.attention=se train.batch_size=2 \
    --tag experiment_se_bs2
```

苹果瘀伤检测是采后质量控制的关键环节。本项目实现了 **MSI-BruiseNet**，一种专为 9 波段近红外多光谱图像（713–920 nm）设计的 UNet 语义分割网络。该架构引入了 **局部光谱异常注意力（LSAA）** 模块，利用局部光谱残差突出瘀伤指示性异常；以及 **ConvGLU** 门控融合机制用于编码器-解码器跳跃连接。模型基于适配 9 通道输入的 MobileNetV2 骨干网络，轻量级设计可在单张 RTX 3060 Laptop GPU（6 GB 显存）上完成训练和推理。代码支持 5 折交叉验证 × 多随机种子、系统性消融实验、按面积分层评估和光谱预分析。

### 图像格式（.npy）

- **路径**：`data/images/`
- **格式**：NumPy `.npy` 文件。`.npy` 是 NumPy 的二进制存储格式，可通过 `np.save()` 创建、`np.load()` 读取。
- **形状**：`(H, W, 9)` — 高度 × 宽度 × 9 个波段
- **数据类型**：`float32`（推荐）或 `uint16`
- **波段**：713, 736, 759, 782, 805, 828, 851, 874, 897, 920 nm
- **命名**：`sample_001.npy`, `sample_002.npy`, ...（数字递增，与掩码一一对应）

**创建示例：**
```python
import numpy as np
# 假设 image 是形状为 (480, 640, 9) 的 float32 数组
np.save("data/images/sample_001.npy", image)
```

### 掩码格式（.npy）

- **路径**：`data/masks/`
- **形状**：`(H, W)` — 与对应图像的高宽完全一致
- **数据类型**：`uint8`
- **像素值**：`0` = 背景，`1` = 瘀伤区域
- **命名**：必须与对应图像文件名完全一致

**创建示例：**
```python
import numpy as np
# 假设 mask 是形状为 (480, 640) 的 uint8 数组，值为 0 或 1
```

## 常见问题 FAQ

### Q: 如何将 TIFF/ENVI 格式转为 .npy？

```python
import numpy as np
# --- TIFF ---
from PIL import Image
# 对于单波段 TIFF，逐波段读取后堆叠
bands = []
for i, tiff_path in enumerate(band_paths):  # 9 个波段的 TIFF 文件路径
    band = np.array(Image.open(tiff_path)).astype(np.float32)
    bands.append(band)
image = np.stack(bands, axis=-1)  # (H, W, 9)
np.save("data/images/sample_001.npy", image)

# --- ENVI ---
# 需要安装 spectral 库: pip install spectral
import spectral
img = spectral.open_image("your_file.hdr")
data = img.load()  # (H, W, num_bands)
# 选择需要的 9 个波段
selected = data[:, :, band_indices]  # band_indices 是长度为 9 的索引列表
np.save("data/images/sample_001.npy", selected.astype(np.float32))
```

### Q: 图像尺寸不一致怎么办？

不用担心！训练时会自动 resize 到 `config.yaml` 中 `data.input_size` 指定的尺寸（默认 512×512）。不同尺寸的原始图像可以混合使用。

### Q: 显存不够（Out of Memory）怎么办？

1. **降低 batch_size**：在 `config.yaml` 中将 `train.batch_size` 从 4 改为 2 或 1
2. **降低输入尺寸**：将 `data.input_size` 从 512 改为 384 或 256
3. **使用混合精度训练**：可以在 `train.py` 中启用 `torch.cuda.amp`（需要手动添加）

```bash
python scripts/train.py --config configs/config.yaml \
    --override train.batch_size=2 data.input_size=384
```

### Q: 没有 GPU 怎么办？

代码支持 CPU 训练，会自动检测设备。但 CPU 训练速度会非常慢（约慢 10-50 倍），建议：
- 减少 `train.epochs` 到 20-50 进行初步验证
- 减少 `data.input_size` 到 256
- 减少 `train.batch_size` 到 1-2
- 使用 Google Colab 免费 GPU 进行完整训练

### Q: 如何快速验证代码是否能跑通？
